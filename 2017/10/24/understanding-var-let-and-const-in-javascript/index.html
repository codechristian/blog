Understanding Var, Let, And Const In JavaScript

October 24, 2017

var

Declaring a variable with the var keyword results in it having function scope.

This means that we are able to use a variable with the var keyword outside of the block it was declared in.

//start of printMessage function scope
function printMessage(message, times) {
  
 //start of for loop block scope
 for (var i = 0; i < times; i++) { 
   
 console.log(message);
   
 }//end of for loop block scope.
  
 //i variable used outside of for loop block it was declared in
 console.log('Value of i is ' + i); 
  
}//end of printMessage function scope

printMessage('hello', 3);

//using i outside of its function scope results in an error
console.log('Value of i is ' + i);

In the above example we are able to declare a var variable i inside a for loop block within the function printMessage. Since we declared i with the keyword var we are able to use it outside of the for loop block.

We will also get an error if we try and use our variable i outside of its function scope.

let

The let keyword will give our declared variables block scope. We will no longer be able to use a let variable outside of its block scope without getting an error.

//start of printMessage function scope
function printMessage(message, times) {
  
 //start of for loop block scope
 for (let i = 0; i < times; i++) { 
   
   console.log(message);
   
 }//end of for loop block scope.
  
 //using i outside of its block scope results in an error
 console.log('Value of i is ' + i); 
  
}//end of printMessage function scope

printMessage('hello', 3);

const

Like let, declaring a variable with the const keyword will restrict that variable with block scope.

//start of printMessage function scope
function printMessage(){
  
  let x = 10;
  
  //start of if statement block scope
  if (x > 0) {
    
    const y = 11;
    
    //const variable used within declared block scope
    console.log(y);
    
  }//end of if statement block scope
  
  //error! const variable used outside block scope
  console.log(y);
  
}//end of printMessage function scope

printMessage();

The difference is that a variable declared with the const keyword cannot be reassigned.

//declare let variable
let x = 10;

//use x variable with let keyword
console.log(x);

//reassigning value of x is ok
x = 9;

//use reassigned x variable with let keyword
console.log(x);

//declared const variable
const y = 11;

//error! once we try and reassign the value of const variable y
y = 12;

If you require that declared variables be able to change (e.g. for loops) then let is the ideal keyword of choice. In any other case using const whenever possible is preferred.
