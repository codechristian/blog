Understanding Var, Let, And Const In JavaScript

October 24, 2017

var

Declaring a variable with the var keyword results in it having function scope.

This means that we are able to use a variable with the var keyword outside of the block it was declared in.

JS Bin on jsbin.com

In the above example we are able to declare a var variable i inside a for loop block within the function printMessage. Since we declared i with the keyword var we are able to use it outside of the for loop block.

We will also get an error if we try and use our variable i outside of its function scope.

let

The let keyword will give our declared variables block scope. We will no longer be able to use a let variable outside of its block scope without getting an error.

JS Bin on jsbin.com

const

Like let, declaring a variable with the const keyword will restrict that variable with block scope.

JS Bin on jsbin.com

The difference is that a variable declared with the const keyword cannot be reassigned.

JS Bin on jsbin.com

If you require that declared variables be able to change (e.g. for loops) then let is the ideal keyword of choice. In any other case using const whenever possible is preffered.
